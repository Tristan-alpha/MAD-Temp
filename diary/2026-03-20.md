## Per-Agent Prompt Optimization

Implemented per-agent prompt optimization for TG-MAD, gated behind `--per_agent_prompts` flag.

### Changes
- **config.py**: Added 3 different seed prompts (`INITIAL_DEBATER_PROMPTS`) for symmetry breaking, tighter 80-word constraint (`OPTIMIZER_CONSTRAINTS_PER_AGENT`), and `PER_AGENT_PROMPTS` default flag.
- **forward_pass.py**: `mad_forward_pass()` now accepts either a single `tg.Variable` (shared mode) or a `list[tg.Variable]` (per-agent mode). Creates one `BlackboxLLM` per agent in per-agent mode. Returns `per_agent_response_vars` (grouped by agent). Skips `tg.sum()` in per-agent mode — caller handles gradient injection.
- **evaluator.py**: Added `create_per_agent_evaluator()` — returns a callable that makes one evaluator LLM call and parses structured `[AGENT_N_FEEDBACK]` output into per-agent feedback strings. Falls back to duplicating full feedback if parsing fails.
- **train.py**: When `--per_agent_prompts` is set: creates 3 independent `tg.Variable`s, 3 `tg.TGD` optimizers, uses manual gradient injection on each agent's last response Variable, then per-agent `backward()` and `optimizer.step()`. Prompt history saves `"prompts"` (list) instead of `"prompt"` (string).
- **evaluate.py**: Detects per-agent format (`"prompts"` key in prompt history), creates list of `tg.Variable`s with `requires_grad=False`, passes to `mad_forward_pass` as list.

### Design rationale
- Different seed prompts break symmetry so agents can differentiate from step 0
- Routed evaluator feedback enables each agent's prompt to receive targeted gradient, not averaged signal
- Manual gradient injection bypasses `tg.sum()` + `TextLoss` overhead, reducing backward context from ~7000 to ~2300 tokens per agent
- 80-word limit (vs previous 500) prevents prompt bloat that was causing response truncation

### Backward compatibility
- All changes gated behind `--per_agent_prompts`; default behavior unchanged
- `evaluate.py` auto-detects format from prompt history file

- All new imports resolve correctly

## Bugfix: Post-review corrections (Codex review)

Fixed 4 issues found by code review:

### P1 fixes (would have crashed at runtime)
1. **Removed `backward(engine=...)` call** — `tg.set_backward_engine()` is called earlier in `train()`, and TextGrad raises `"Both an engine is provided and the global engine is set"` if you also pass `engine=` to `backward()`. The per-agent path would have crashed on the first sample.
2. **Replaced response-var gradient injection with direct prompt-var injection** — `Variable.backward()` clears `self.gradients = set()` on the root variable before traversal (variable.py:174), so any manually-added gradients on `last_resp` were being discarded. Now we inject feedback directly onto each `prompt_i` Variable, and `TGD.step()` reads it via `parameter.get_gradient_text()`. This also eliminates 3 backward-engine LLM calls per sample.

### P2 fixes (edge cases)
3. **Cycle seed prompts for n_agents > 3** — `INITIAL_DEBATER_PROMPTS[i]` → `INITIAL_DEBATER_PROMPTS[i % len(INITIAL_DEBATER_PROMPTS)]` so `--n_agents 4` doesn't `IndexError`.
4. **Validate parsed agent marker IDs** — `_parse_per_agent_feedback` now checks `all((i+1) in feedbacks for i in range(n_agents))` instead of `len(feedbacks) >= n_agents`. Prevents `KeyError` when evaluator emits non-sequential agent IDs (e.g. sections 1,3 but skips 2).

### Validation
- Parser unit tests pass: normal case, missing-agent fallback, and n_agents > 3 fallback
- All files pass syntax check and imports resolve

## Bugfix: Second review round (Codex)

### P1: optimizer.step() was inside the sample loop
- With `batch_size > 1`, each sample triggered `opt.step()` + `opt.zero_grad()`, so the 2nd sample in a batch used already-mutated prompts. Moved `opt.step()` and `opt.zero_grad()` outside the sample loop so gradients accumulate across all batch samples before a single update.

### P2: Hardcoded 3-agent feedback markers in evaluator prompt
- `create_per_agent_evaluator()` accepted `n_agents` but the prompt only listed `[AGENT_1_FEEDBACK]` through `[AGENT_3_FEEDBACK]`. Made markers dynamic: generates `[AGENT_{i+1}_FEEDBACK]` for `i in range(n_agents)`. Verified with `n_agents=4`.
